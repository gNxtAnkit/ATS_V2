# 30 — Dev Team Execution Plan

## Suggested squads

Platform Foundations, Identity/Security, Tenant/Admin, ATS Domain, Agency/Portal, Automation/AI, Monetization/Insights, Compliance/SRE, Frontend Platform.

## Ticket template

Every ticket must include requirement IDs, owning service, tables touched, APIs, events, permissions, RLS/tenant notes, audit requirements, config keys, tests, rollout/rollback, and UI states where applicable.

## Definition of ready

API/event contracts, table ownership, permissions, UI states, and tests are clear.

## Definition of done

Code, migrations, OpenAPI, event catalog, tests, docs, audit, telemetry, frontend states, and rollback notes complete.

## Governance

Architecture review for cross-service dependencies, security review for auth/RLS/secrets, migration review for DB changes, AI governance review for autonomy/prompt changes, billing review before charging.
