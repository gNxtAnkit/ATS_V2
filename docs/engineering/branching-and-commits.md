# Branching and Commit Standards

## Branches

Use short, descriptive branch names:

- `phase-0/repo-foundation`
- `phase-1/db-baseline`
- `identity/session-foundation`

## Commits

Commits should be reviewable and scoped to one coherent change.

Recommended prefixes:

- `infra:`
- `docs:`
- `test:`
- `db:`
- `service:`
- `frontend:`

Do not mix unrelated formatting, refactors, dependency upgrades, and behavior changes in one commit.
